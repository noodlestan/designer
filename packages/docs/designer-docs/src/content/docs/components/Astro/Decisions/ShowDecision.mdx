```ts
import { ShowDecision } from '@noodlestan/designer-shows/astro';
```

This component renders the appropriate visualization according to the decision type.

In this example we are rendering decisions of 4 different types:

<CardGrid>
    <FlexStack>
        <label>Color</label>
        <ShowDecision store={store} d="Action Color" size="m" />
    </FlexStack>
    <FlexStack>
        <label>Space</label>
        <ShowDecision store={store} d="Space Unit" size="m" />
    </FlexStack>
    <FlexStack>
        <label>Color Set</label>
        <ShowDecision store={store} d="Heat Map Color Set" size="s" />
    </FlexStack>
    <FlexStack>
        <label>Space Scale</label>
        <ShowDecision store={store} d="Layout Space Scale" size="s" />
    </FlexStack>
</CardGrid>

## Props

### ༶ [DecisionProps](/components/Astro/Props): `store` and `d`

These two **mandatory** props serve to identify and retrieve the decision.

### ༶ [ValueProps](/components/Astro/Props#valueprops): `value`, `viz`, and `size`

You can use these props to control how the visualization and value are displayed.

In the following example we are hiding the visualization and customizing
the color value display to render both `rgb` and `hsl` color spaces.

```mdx
<ShowDecision store={store} d="Action Color" viz={false} value={['rgb', 'oklch']} />
```

<CardGrid>
    <ShowDecision store={store} d="Action Color" viz={false} value={['rgb', 'oklch']} />
</CardGrid>

In this other example we are rendering the space scale with size `xs`.

```mdx
<ShowDecision store={store} d="Layout Space Scale" size="xs" />
```

<CardGrid>
    <ShowDecision store={store} d="Layout Space Scale" size="xs" />
</CardGrid>

the different ways in which decision values can be displayed,
refer to the documentation for specific [Decision Components](/components/Astro/#decisions).
