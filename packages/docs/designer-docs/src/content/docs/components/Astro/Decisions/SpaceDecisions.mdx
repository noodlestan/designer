## Components

These components render space related decisions.

```ts
import { ... } from '@noodlestan/designer-shows/astro';
```

### ShowSpaceValueDecision

Renders a [SpaceValueDecision](/decision-models/space-value).

Use the [viz prop](#-viz-true-boolean--spacevizname) to chose the visualization type.

```mdx
<ShowSpaceValueDecision store={store} d="Space Unit" size="s" />
```

<CardGrid>
    <ShowSpaceValueDecision store={store} d="Space Unit" size="s" />
</CardGrid>

### ShowSpaceScaleDecision

Renders a [SpaceScaleDecision](/decision-models/space-scale).

Use the [viz prop](#-viz-true-boolean--spacevizname) to chose the visualization type.

```mdx
<ShowSpaceScaleDecision store={store} d="Sizing Space Scale" size="xs" />
```

<CardGrid>
    <ShowSpaceScaleDecision store={store} d="Sizing Space Scale" size="xs" />
</CardGrid>

## Props

### ༶ [DecisionProps](/components/Astro/Props): `store` and `d` (mandatory)

These two **mandatory** props serve to identify and retrieve the decision.

### ༶ [ValueProps ༶ value](/components/Astro/Props/#-value-true-boolean--string--object)

The `value` prop controls how the decision value is displayed.

At the moment it has no effect on Space Decisions.

### ༶ [ValueProps ༶ size](/components/Astro/Props#-size-auto-showvaluesize)

The `size` optional prop controls the size of the visualization.

Refer to [ShowValueSize](/api/designer-shows/Types/ShowValueSize) for available sizes.

### ༶ [ValueProps ༶ viz](/components/Astro/Props/#-viz-true-boolean--string)

Chose visualization type.

In this example we set the visualization to `circle`.

```mdx
<ShowDecision store={store} d="Avatar Sizes" viz="circle" />
```

<CardGrid>

<ShowDecision store={store} d="Avatar Sizes" viz="circle" />
</CardGrid>

Refer to [SpaceVizName](/api/designer-shows/Types/SpaceVizName) for available visualizations.

### ༶ [ValueProps ༶ options](/components/Astro/Props/#-options-optional-object)

Configure the visualization.

At the moment it has no effect on Space Decisions.
