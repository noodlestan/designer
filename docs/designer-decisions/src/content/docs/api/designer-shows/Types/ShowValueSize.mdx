Available visualization sizes for [all value components](/components/Astro/Values).

```ts
type ShowValueSize = 'xs' | 's' | 'm' | 'l' | 'xl' | 'auto';
```

When rendering decisions you can chose a different visualization size
setting the `size` [prop](/components/Astro/Props#valueprops) to one of these.

```mdx
<ShowDecision store={store} d="Action Color" size={...} />
```

## Type Values

### `xs`

```mdx
<ShowDecision store={store} d="Action Color" size={'xs'} />
```

### `auto` (default)

Defaults to `auto` in which case it stretches to use all available space.

```mdx
<ShowDecision store={store} d="Action Color" />
```

<FlexStack vpad>
    <ShowDecision store={store} d="Action Color" />
</FlexStack>

In these cases, consider adding an explicit `size` value, or composing the decisions within
some sort of grid or list layout.

```mdx
<FlexRow>
    <ShowDecision store={store} d="Action Color" />
    <ShowDecision store={store} d="Brand Blue" />
    <ShowDecision store={store} d="Brand Pink" />
</FlexRow>
```
