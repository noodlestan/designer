These components render a decision's visualization directly.

<Aside type="tip">
    You don't have to use these components directly in your docs. Using
    [ShowDecision](/components/Astro/ShowDecision/) will select the appropriate one for you.
</Aside>

They are exposed so that anyone can [create custom visualizations](/guides/creating-custom-design-decision-visualizations/), composing values and visualizations directly in their custom components.

Refer to the table on [Composable Types](/components/Astro/Composable/Types) for all decision types and their corresponding visualizations.

## Common Props

### à¼¶ `v`: BaseValue&lt;unknown&gt;

A decision value of the appropriate type.

### à¼¶ [VizProps](/components/Astro/Props#vizprops): `viz`, `size`, and `options`

These props control how the visualization should be displayed.

Decision components, such as `<ShowDecision/>` expose them well, so that you can have full control over visualization options.

<Aside type="note">
    In the `<ShowDecision />` and `<ShowDecisionCard />` components the `viz` and `options` props
    are typed to base types and will not auto-complete on your IDE. You can use the types listed
    below to improve your experience and ensure type safety.
</Aside>

### ShowColorChannelViz

#### à¼¶ `v`: [ColorChannelValue](/api/designer-decisions/Types/Values/ColorChannelValue)

#### à¼¶ `viz`: -

This prop has no effect on this component.

It always renders the [Color Swatch](/api/designer-shows/Types/ColorVizName#swatch-default) visualization, providing defaults for the complimentary channels.

#### à¼¶ `channel`: [ColorChannelName](/api/designer-decisions/Types/Primitives/ColorChannelName)

This mandatory prop informs the visualization of the type of value.

This is used to determine which complimentary channels are necessary to compose a color for the color swatch.
