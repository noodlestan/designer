```ts
import { ShowDecision } from '@noodlestan/designer-shows/astro';
```

Renders the appropriate visualization according to the decision type.

In the following two examples we use the `<ShowDecision/>` to render two different
decision types.

A [color value](/decision-models/color-value) decision:

```mdx
<ShowDecision store={store} d="Action Color" size={'s'} viz="bg" />
```

<ShowDecision store={store} d="Action Color" size={'s'} viz="bg" />

A [space scale](/decision-models/space-scale) decision:

```mdx
<ShowDecision store={store} d="Sizing Space Scale" size={'xs'} />
```

<ShowDecision store={store} d="Sizing Space Scale" size={'xs'} />

## Props

### ༶ [DecisionProps](/components/Astro/Props): `store` and `d`

These two **mandatory** props serve to identify and retrieve the decision.

### ༶ [ValueProps](/components/Astro/Props#valueprops): `size`, `value`, `viz`, and `options`

These props control how the visualization and value are displayed.

The [Rendering Decisions in Docs](/guides/documenting-design-decisions-with-astro-and-mdx/) guide
showcases these props in action.

Refer to each [Value Component](/components/Astro/Composable/Values) for details
on available visualizations and options.
