```ts
import { ShowDecision } from '@noodlestan/designer-shows/astro';
```

Renders the appropriate visualization according to the decision type.

In the following two examples we use the `<ShowDecision/>` to render two different
decision types.

A [color value](/decision-models/color-value) decision:

```mdx
<ShowDecision store={store} d="Action Color" size={'s'} viz="bg" />
```

<ShowDecision store={store} d="Action Color" size={'s'} viz="bg" />

A [space scale](/decision-models/space-scale) decision:

```mdx
<ShowDecision store={store} d="Sizing Space Scale" size={'xs'} />
```

<ShowDecision store={store} d="Sizing Space Scale" size={'xs'} />

## Composes

This component composes a different value and visualization component depending on the decision type.

Refer to the table on [Composable Types](/components/Astro/Composable/Types) to learn about the options available.

## Props

### ༶ [DecisionProps](/components/Astro/Props): `store` and `d`

These two **mandatory** props serve to identify and retrieve the decision.

### ༶ [ValueProps](/components/Astro/Props#valueprops): `value`

This prop control whether the value should be shown and how to format it.

Refer to each [Value Component](/components/Astro/Composable/Values) for details on available formats.

### ༶ [VizProps](/components/Astro/Props#vizprops): `viz`, `size`, and `options`

These props control whether the visualization should be shown, which specific visualization to show, and how to customize it.

Refer to each [Visualization Component](/components/Astro/Composable/Visualizations) for details
on available visualizations and options.
