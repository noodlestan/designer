Renders a card layout with the appropriate visualization
according to the decision type, along with any decision details you might want to display.

By default it renders only the decision **description** and **usage** guidelines.

<FlexStack vpad>
    <ShowDecisionCard store={store} d="Action Color" />
</FlexStack>

## Composes

This component composes:

-   [ShowDecision](/components/Astro/ShowDecision) component (value and visualization)
-   inside of `DecisionCardLayout` (ðŸš§ link)
-   and one or more [Decision Attribute](/components/Astro/Composable/Attributes).

## Props

### à¼¶ [DecisionProps](/components/Astro/Props): `store` and `d`

These two **mandatory** props serve to identify and retrieve the decision.

### à¼¶ [ValueProps](/components/Astro/Props#valueprops): `value`, `viz`, and `options`

You can use these props to control how the visualization and value are displayed.

Note: ðŸš§ The common [size](/components/Astro/Props/#-size-auto-showvaluesize) prop is not exposed
by `<ShowDecisionCard />` and the size is managed internally.

In this example we are rendering a [ColorValueDecision](/decision-models/color-value)
and at the same time:

-   hiding the visualization;
-   displaying its value in `rgb` and `hsl` formats.

```mdx
<ShowDecisionCard store={store} d="Action Color" viz={false} value={['rgb', 'oklch']} />
```
