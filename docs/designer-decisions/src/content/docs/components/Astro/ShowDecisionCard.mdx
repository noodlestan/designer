Renders a card layout with the appropriate visualization according to the decision type, along with any decision details you might want to display.

By default it renders only the decision **description** and **usage** guidelines.

<ShowDecisionCard store={store} d="Action Color" />

## Composes

This component composes:

-   [ShowDecision](/components/Astro/ShowDecision) component (value and visualization)
-   inside of `DecisionCardLayout` (ðŸš§ link)
-   and one or more [Decision Attribute](/components/Astro/Composable/Attributes).

## Props

### à¼¶ [DecisionProps](/components/Astro/Props): `store`, `d` and `layout`

These two **mandatory** `store` and `d` props serve to identify and retrieve the decision.

The optional `layout` prop controls the flow of multiple items and the value/viz pair.

### à¼¶ [ValueProps](/components/Astro/Props#valueprops): `value`

This prop control whether the value should be shown and how to format it.

Refer to the table on [Composable Types](/components/Astro/Composable/Types) to learn which `value` options are available for each decision type.

### à¼¶ [VizProps](/components/Astro/Props#vizprops): `viz`, `size`, and `options`

These props control whether the visualization should be shown, which specific visualization to show, and how to customize it.

In this example we are rendering a [ColorValueDecision](/decision-models/color-value), hiding its value and increasing the size of the color swatch.

```mdx
<ShowDecisionCard store={store} d="Action Color" value={false} size="l" />
```

Refer to the table on [Composable Types](/components/Astro/Composable/Types) for details on available visualizations and options.
